---
title: "Criar Pagamento PIX"
api: "POST /payment/create"
description: "Cria uma nova transação PIX e gera o QR Code e o código Copia e Cola."
---

Este é o endpoint principal para receber pagamentos. Ele gera um QR Code dinâmico que pode ser lido por qualquer aplicativo bancário brasileiro.

### Parâmetros do Corpo (JSON)

<ParamField body="value" type="number" required>
  Valor em reais (ex: 100.50). O valor mínimo permitido é **R$ 1,00**.
</ParamField>

<ParamField body="description" type="string">
  Texto que aparecerá no extrato do cliente e no seu painel Zynx. Máximo de 255 caracteres.
</ParamField>

<ParamField body="coverFee" type="boolean" default="false">
  Define quem paga a taxa de R$ 0.50:
  - `true`: A taxa é somada ao valor total (o cliente paga a taxa).
  - `false`: A taxa é descontada do valor recebido (você paga a taxa).
</ParamField>

<ParamField body="splitUser" type="string">
  E-mail de outro usuário Zynx cadastrado para receber uma parte do valor automaticamente.
</ParamField>

<ParamField body="splitTax" type="number">
  Porcentagem (0.01 a 100) do valor que será enviado para o `splitUser`.
</ParamField>

### Entendendo a Resposta

A Zynx retorna valores monetários em **centavos** para garantir precisão total.

<ResponseField name="success" type="boolean">
  Indica se a transação foi criada com sucesso no nosso gateway.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Propriedades do Objeto">
    <ResponseField name="id" type="string">
      ID único da transação na Zynx (ex: `pay_abc123`). Use este ID para consultas futuras.
    </ResponseField>
    <ResponseField name="valueInReais" type="number">
      O valor exato que o cliente deve pagar no banco.
    </ResponseField>
    <ResponseField name="netValue" type="number">
      O valor líquido que entrará no seu saldo (em centavos).
    </ResponseField>
    <ResponseField name="fee" type="number">
      O valor da taxa aplicada (em centavos).
    </ResponseField>
    <ResponseField name="qrcodeUrl" type="string">
      String Base64 da imagem do QR Code. Pode ser usada diretamente em uma tag `<img src="..." />`.
    </ResponseField>
    <ResponseField name="copyPaste" type="string">
      Código PIX Copia e Cola para usuários que estão no celular.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST 'https://api.zynxwallet.com.br/api/v1/payment/create' \
  -H 'X-API-Key: zynx_sua_chave_aqui' \
  -H 'Content-Type: application/json' \
  -d '{
    "value": 150.00,
    "description": "Venda de Produto #123",
    "coverFee": true,
    "splitUser": "parceiro@zynx.com",
    "splitTax": 10.0
  }'
```
</RequestExample>

<Tip>
  **Dica de Implementação:** Sempre salve o `id` retornado em seu banco de dados associado ao pedido do seu cliente. Isso facilitará a conciliação quando o webhook de confirmação chegar.
</Tip>
