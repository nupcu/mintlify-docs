A API da Zynx Wallet utiliza chaves de API para autenticar requisições. Você pode gerenciar suas chaves através do [Dahsboard](https://zynxwallet.com.br/dashboard/credentials).

### Cabeçalho de Autenticação

Todas as requisições para a API devem incluir o cabeçalho `X-API-Key`.

| Header | Valor | Descrição |
| :--- | :--- | :--- |
| `X-API-Key` | `zynx_...` | Sua chave de API secreta. |

<Warning>
  **Segurança em Primeiro Lugar:** Nunca exponha sua chave de API em ambientes públicos, como repositórios do GitHub ou no código front-end (JavaScript do navegador).
</Warning>

### Exemplo de Requisição

```bash
curl -X GET "https://api.zynxwallet.com.br/api/v1/user/balance" \
  -H "X-API-Key: zynx_sua_chave_aqui"
```

### Erros de Autenticação

Se a sua chave for inválida, expirada ou estiver ausente, a API retornará:

- **401 Unauthorized:** A chave de API é inválida ou não foi fornecida.
- **403 Forbidden:** A chave de API não tem permissão para acessar o recurso solicitado.

<Tip>
  Se você suspeitar que sua chave foi comprometida, gere uma nova imediatamente no painel e revogue a antiga.
</Tip>
