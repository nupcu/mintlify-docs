## Configurar Webhook

Configure webhooks para receber notificações automáticas em tempo real sobre eventos importantes, como a confirmação de um pagamento ou o processamento de um saque.

### Requisição

<ParamField body="url" type="string" required>
  URL do seu servidor que receberá as notificações (POST).
</ParamField>

<ParamField body="events" type="array" required>
  Lista de eventos para monitorar (ex: `["payment.completed", "withdrawal.created"]`).
</ParamField>

<RequestExample>

```bash cURL
curl --request POST \
  --url https://api.zynxwallet.com.br/api/v1/webhooks \
  --header 'Content-Type: application/json' \
  --header 'X-API-Key: zynx_sua_chave' \
  --data '{
  "url": "https://seusite.com/webhook",
  "events": ["payment.completed", "withdrawal.created"]
}'
```

</RequestExample>

<ResponseExample>

```json Exemplo de Resposta
{
  "success": true,
  "message": "Webhook configurado com sucesso"
}
```

</ResponseExample>

## Estrutura do Payload

Quando um evento ocorre, a Zynx Wallet enviará um `POST` para sua URL com o seguinte formato:

<ResponseField name="event" type="string">
  O tipo do evento ocorrido.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Detalhes do Evento">
    <ResponseField name="id" type="string">
      ID da transação relacionada.
    </ResponseField>
    <ResponseField name="status" type="string">
      Novo status da transação (`COMPLETED`, `FAILED`, etc).
    </ResponseField>
    <ResponseField name="value" type="number">
      Valor da transação em centavos.
    </ResponseField>
  </Expandable>
</ResponseField>

### Exemplo de Notificação

```json
{
  "event": "payment.completed",
  "timestamp": "2024-02-16T15:30:00Z",
  "data": {
    "id": "pay_abc123",
    "status": "COMPLETED",
    "value": 15300,
    "netValue": 15000,
    "updatedAt": "2024-02-16T15:05:00Z"
  }
}
```

### Eventos Suportados

| Evento | Descrição |
| :--- | :--- |
| `payment.completed` | Um pagamento PIX foi confirmado. |
| `payment.failed` | Um pagamento PIX expirou ou falhou. |
| `withdrawal.created` | Um novo saque foi solicitado. |
| `withdrawal.completed` | Um saque foi processado com sucesso. |
| `withdrawal.failed` | Um saque foi rejeitado ou falhou. |
